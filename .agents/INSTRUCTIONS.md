# Repository Guidelines

## Project Overview

This is a pnpm monorepo of Notion Workers built with `@notionhq/workers`. Each worker in `workers/` is a standalone package that extends Notion agents with third-party API integrations.

## Project Structure & Module Organization

```
notion-workers/
├── package.json            # Root workspace config (pnpm)
├── pnpm-workspace.yaml     # Defines workers/* as workspace packages
├── tsconfig.base.json      # Shared TypeScript configuration
└── workers/
    ├── mercury-invoices/   # Mercury banking invoices worker
    │   ├── .env.example    # Required environment variables template
    │   ├── package.json    # Worker package config
    │   ├── tsconfig.json   # Worker TypeScript config
    │   └── src/
    │       ├── index.ts    # Worker + tool definitions
    │       ├── mercury.ts  # Mercury API client
    │       └── types.ts    # TypeScript interfaces
    ├── toggl-time/         # Toggl time tracking worker
    │   ├── .env.example
    │   ├── package.json
    │   ├── tsconfig.json
    │   └── src/
    │       ├── index.ts        # 4 tool definitions
    │       ├── toggl.ts        # TogglAPIClient class
    │       ├── types.ts        # Toggl + report interfaces
    │       ├── formatters.ts   # Time/money/report formatting
    │       └── reports.ts      # Report building logic
    └── time-calc/          # Time calculation utilities worker
        ├── package.json
        ├── tsconfig.json
        └── src/
            └── index.ts    # 3 tool definitions + utility functions
```

- `workers.json` is generated by the CLI (`ntn workers new`) and is gitignored — it contains deployment-specific workspace and worker IDs.
- `.env` files are gitignored — they contain API secrets. Use `.env.example` as a template.

## Worker & Capability API (SDK)

`@notionhq/workers` provides the `Worker` class; the `ntn` CLI handles deployment and execution.

### Defining tools

```ts
import { Worker } from "@notionhq/workers";

const worker = new Worker();
export default worker;

worker.tool("toolName", {
  title: "Human-readable Title",
  description: "Description shown to the agent",
  schema: {
    type: "object",
    properties: { /* JSON Schema */ },
    required: ["..."],
    additionalProperties: false,
  },
  execute: async (input) => {
    // Return a string result
    return JSON.stringify({ result: "..." });
  },
});
```

Each `tool` becomes a callable function for Notion agents:
- `title` and `description` appear in the Notion UI and help the agent understand the tool.
- `schema` specifies what data the agent must supply (JSON Schema).
- `execute` receives validated input and returns a string.

## Setup Guide (for helping users get started)

### Prerequisites
- Node.js >= 22
- pnpm v10+
- Notion CLI: `npm i -g ntn`

### First-time setup

1. Clone and install:
   ```sh
   git clone https://github.com/dawn-patrol-llc/notion-workers.git
   cd notion-workers
   pnpm install
   ```

2. Log in to Notion:
   ```sh
   ntn login
   ```

3. Navigate to the worker directory and initialize:
   ```sh
   cd workers/mercury-invoices
   ntn workers new
   ```
   This creates `workers.json` linking the worker to the user's Notion workspace.

4. Configure environment variables:
   ```sh
   cp .env.example .env
   ```
   Then fill in the values in `.env`.

5. Deploy:
   ```sh
   ntn workers deploy
   ```

6. Push secrets to the deployed worker:
   ```sh
   ntn workers env push
   ```

7. Add the worker's tools to a Notion agent in the Notion UI.

## Workers Reference

### toggl-time

Integrates with the [Toggl Track API](https://developers.track.toggl.com/) for time tracking and billable hours reporting.

**Environment variables:**
| Variable | Description |
|----------|-------------|
| `TOGGL_API_KEY` | Toggl API token (from https://track.toggl.com/profile) |

**Tools:**
| Tool | Purpose |
|------|---------|
| `listWorkspaces` | List available Toggl workspaces (id, name, currency). |
| `listClients` | List non-archived clients in a workspace. |
| `monthlyInvoice` | Generate a monthly invoice report with billable hours, rates, and amounts broken down by client and person. |
| `billableSummary` | Get a quick billable summary for any date range. |

**Key implementation details:**
- `src/toggl.ts` — `TogglAPIClient` class wrapping Toggl API v9 and Reports API v3. Auth is Basic with API token.
- `src/reports.ts` — `buildInvoiceReport()` and `buildBillableSummary()` aggregate time entries by client/person.
- `src/formatters.ts` — Formatting utilities for time, money, and report output (text/markdown/json).

### time-calc

Time format conversion and billable amount calculation utilities. No external API dependencies.

**Tools:**
| Tool | Purpose |
|------|---------|
| `convertToDecimal` | Convert HH:MM:SS to decimal hours, optionally calculate billable amount with a rate. |
| `convertToHms` | Convert decimal hours to HH:MM:SS. |
| `calculateBillable` | Given decimal hours and hourly rate, return billable amount. |

### mercury-invoices

Integrates with the [Mercury Accounts Receivable API](https://docs.mercury.com/reference/accounts-receivable).

**Environment variables:**
| Variable | Description |
|----------|-------------|
| `MERCURY_API_TOKEN` | Mercury API bearer token |
| `MERCURY_ACCOUNT_ID` | UUID of the Mercury account to receive payments |

**Tools:**
| Tool | Purpose |
|------|---------|
| `createInvoice` | Creates a draft invoice with line items, due date, service period. Returns invoice ID and payment link. |
| `getInvoice` | Retrieves invoice details by UUID. |
| `listCustomers` | Lists customers with optional name search. Use to find customer UUIDs. |
| `listInvoices` | Lists invoices filtered by customer name and/or status (Unpaid, Paid, Cancelled, Processing). |

**Key implementation details:**
- `src/mercury.ts` — All API calls go through `https://api.mercury.com/api/v1/ar/`. Auth is via `Authorization: Bearer {token}` header.
- Invoices are created with `sendEmailOption: "DontSend"` (drafts only, never auto-sent).
- Payment URLs are built from the invoice slug: `https://app.mercury.com/invoices/{slug}`.
- Invoice numbers auto-generate as `INV-{YYYY-MM-DD}` when not provided.

## Build, Test, and Development Commands

```sh
# Install all dependencies
pnpm install

# Type-check all workers
pnpm check

# Build all workers
pnpm build

# Deploy a specific worker
cd workers/mercury-invoices && ntn workers deploy

# Test a tool locally (uses .env automatically)
ntn workers exec createInvoice --local -d '{"customerId": "...", ...}'

# Test against the deployed worker (requires deploy + env push first)
ntn workers exec createInvoice -d '{"customerId": "...", ...}'

# Push local .env secrets to deployed worker
ntn workers env push

# Set a single secret on the deployed worker
ntn workers env set MERCURY_API_TOKEN=your-token
```

## Debugging & Monitoring Runs

```sh
# List recent runs
ntn workers runs list

# Get logs for a specific run
ntn workers runs logs <runId>

# Get logs for the latest run
ntn workers runs list --plain | head -n1 | cut -f1 | xargs -I{} ntn workers runs logs {}
```

The `--plain` flag outputs tab-separated values for easy piping.

## Adding a New Worker

1. Scaffold:
   ```sh
   cd workers
   ntn workers new
   ```

2. Add the worker directory to the pnpm workspace (already covered by `workers/*` glob in `pnpm-workspace.yaml`).

3. Create a `.env.example` listing required environment variables with placeholder values.

4. Define tools in `src/index.ts` following the pattern in `mercury-invoices`.

5. Add a `README.md` to the worker directory.

6. Deploy with `ntn workers deploy`.

## Coding Style & Conventions

- TypeScript with `strict` enabled.
- Explicit types for all API request/response shapes (no `any`).
- Environment variables accessed via `process.env` with null checks that throw descriptive errors.
- Tool execute functions return `JSON.stringify(...)` results.
- Use pnpm (not npm) for all package management.
